適用對象新小伙伴、企業 AI 助理、專案入口網站

這個技能做什麼

partner-superpower-portal 用來幫每位小伙伴建立自己的「超能力入口網站」。固定包含主入口、所有網址列表、HTML PPT 目錄、技能列表、Google Drive 檔案總管；也可以加客服入口、AI 整合網站與專題分析報告。

小伙伴要準備的帳號

自己的 Gmail

用來申請與管理 Google Drive、GitHub、OpenAI 或其他服務。不要使用其他小伙伴的 Gmail。

自己的 GitHub

用自己的 GitHub 帳號開 repository，GitHub Pages 網址格式通常是 https://帳號.github.io/倉庫名稱/。

自己的 Google Drive

公開交付檔案放在自己的 Drive 資料夾，只有確認可公開的檔案才放進檔案總管。

自己的公開連結

官方網站、社群、LINE/Telegram 客服、AI 系統與簡報連結，都必須是該小伙伴或該公司自己的。

標準入口網站功能

  1. 主入口：第一頁顯示小伙伴名稱與所有功能方塊。
  2. 所有網址列表：整理官網、社群、客服、AI 系統與公開工具。
  3. HTML PPT 目錄：把公開簡報依日期由新到舊排列。
  4. 技能列表：列出小伙伴目前能做的事情與未來可擴充能力；其他小伙伴的客戶會看得到，所以只放技能名稱、簡短介紹與狀態，不放詳細教學或內部流程。
  5. 檔案總管：整理 Google Drive 公開檔案、檔名、說明、日期、預覽與下載鏈結。

技能列表公開程度

文文自己的技能列表

文文入口網站是給 Jason 看的，Jason 是文文與小伙伴的唯一主人，所以可以也必須放完整詳細資料、技能分享、完整文件、版號、產生日期、管理資訊與教學內容。

其他小伙伴的技能列表

入口網站主要給客戶看、幫助客戶理解服務與處理業務，所以只放名稱、簡短用途、狀態與大分類。不要放完整教學、內部步驟、腳本、Prompt、Repository 細節或可被客戶直接學走的流程。

客戶可見內容保留原則

客戶可見頁面要呈現「能幫客戶完成什麼」，不要呈現「內部怎麼做到」。所有教小伙伴的技能與方法都屬於 Jason 團隊的內部能力資產；小伙伴可以學會怎麼服務客戶，但任何對外環境、客戶入口、公開文件、客服回覆、簡報或網站，都不得洩漏實作方法、內部流程、Prompt、腳本、架構、部署方式、完整 SOP 或可複製的操作細節。對外只能談應用、成果、服務項目與客戶價值。

不能放的內容

不要把 API Key、Token、密碼、驗證碼、私鑰、付款資料、客戶內部資料、私人 Drive 檔案、後台管理網址或其他小伙伴的帳號放進公開網站。這個入口網站只放「確定可以公開給別人看的資料」。

學習使用方式

  1. 先準備自己的 Gmail、GitHub repository、Google Drive 公開資料夾。
  2. 把公司或小伙伴名稱、公開網址、簡報、技能、檔案清單整理成 JSON 設定。
  3. 用 partner-superpower-portal 產生靜態網站。
  4. 把產生出的檔案放到自己的 GitHub Pages repository。
  5. 發布後，把主入口網址交給 Jason 或文文檢查。

給小伙伴的簡短指令可以這樣寫：請使用 partner-superpower-portal，幫我用自己的 Gmail、GitHub、Google Drive 建立我的超能力入口網站。

技能檔案結構

  • SKILL.md：技能觸發說明、核心流程、隱私規則、設計標準與驗證方式。
  • references/function-map.md：三個既有小伙伴入口網站的功能拆解，以及通用模板規格。
  • assets/sample-config.json：小伙伴自己的 Gmail、GitHub、Google Drive、網址、簡報、技能、檔案清單設定範本。
  • scripts/build_portal.py：從 JSON 設定產生 GitHub Pages 靜態入口網站的工具。
